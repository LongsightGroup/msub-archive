Roles and Permissions in Project Sites

Project sites contain a set of pre-defined roles that users are given when a site is created. Each role has a slightly different set of permissions associated with it. These permissions determine what a user in each role can do with the various tools.

Roles and permissions vary depending on institutional requirements. Most often, however, project sites have roles such as Organizer, Participant, Guest, and Tech Support. Unlike course sites, project sites often require more open collaboration among peers, which may mean giving all site participants the same role or expanding the permissions in certain tools for roles on your site.

Note: For an overview of roles and permissions, see Roles and Permissions in the CLE.

It is usually necessary to add users manually to project sites. You can add users through either the Site Setup tool, located in My Workspace, or through the Site Editor tool, located in the Administer category of your project site. In RSN, the Roster Import tool is also available for this purpose. Using this tool, you can bulk-add a group of users to your site via an uploaded list.

You may change a user’s role at any time through the Site Editor tool.
