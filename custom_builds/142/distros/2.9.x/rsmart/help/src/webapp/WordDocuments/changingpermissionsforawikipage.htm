Changing Permissions for a Wiki Page

Warning: If a group has lower permissions for using the tool as a whole than for this page, the overall tool permissions are applied. Therefore, if a role does not have edit permissions for the Wiki tool as a whole, you cannot enable users in that role to edit a specific page in the wiki.

To edit permissions for a single page in the wiki, access that page and follow these steps:

1.  Click the Info button on the menu bar. The system displays the Info screen for this page.

2.  Referring to the table below, check and un-check boxes to switch permissions off and on for access to this page.  

Permission

Description

Read

Users are allowed to view this wiki page.

Edit

Users are allowed to change this wiki page.

Admin

Users are allowed to change site permissions.

Note: Disabling Admin permissions causes errors when you try to access the wiki. If you disable these permissions, contact your system administrator and request that the permissions be reset.

You may set permissions in the following ways:

•      To enable or disable site permissions on the page, check or uncheck the appropriate boxes for Enable/Disable on this page.

•      To set permissions for the user who created the page, check or uncheck the appropriate boxes for Page Owner.

•      To allow users in the CLE who are not members of your site to view/edit the page, check the appropriate boxes for Authenticated users. Keep in mind that when you check the “Read” box, the CLE automatically displays “yes” in the “Read” column for Anyone at all, and the HTML (public view) version of the page may be viewed without logging in to the CLE.

3.  To save your changes, click Save.
