Configuring a Basic LTI Tool

When you first access the Basic LTI tool in a site, the CLE displays a message that the tool has not yet been configured. To configure the tool, follow these steps:

1.  Click on the edit icon () in the far right corner. The CLE displays the following page.

2.  Complete the Required Information section.

Note: Keep in mind that you must obtain the information for this section, and that the information will be unique for each external solution.

From the dropdown menu, select your method for providing the necessary information to launch the external solution:

•      If you have the URL for the external solution, select URL and Password and enter the URL in the displayed field.

•      If you have an XML descriptor for the external solution, copy the descriptor, select XML Paste from the menu, and paste the descriptor in the displayed box.

Next, enter the appropriate information in the Remote Tool Key and Remote Tool Secret fields.

3.  Complete the Display Information section.

a.  In the Set Button Text field, enter the name you want displayed for the tool in the site tool menu. If you already renamed the tool when you added it to the site, you can skip this step.

b.  In the Set Tool Title field, enter the name you want displayed to users when they access the tool. (Note: if you leave this field blank, IMS Basic Learning Tool Interoperability will display.)   

4.  If you want scores from the external solution sent to Gradebook/Gradebook2, create a gradebook item for the scores by entering the appropriate information in the Create Gradebook Item field. This step is optional.

5.  Configure optional launch settings for the external solution.

a.  Check the box for Open in a New Window if you want the solution to open outside the site in a new tab or window.

b.  Check the box for Maximize Window Width if the solution is opening within the site and you want the site’s tool menu to collapse automatically when users select the tool (note: depending on how your CLE instance is configured, this feature may not be applicable; for more information, see Collapsing the Tool Menu).

c.  If the solution is opening within the site, set the height of the viewing window by entering the appropriate size, in pixels, in the iFrame Height field.

d.  Check the box for Debug Launch if you would like the CLE to display launch data before the solution launches. (note: all site members will see this display each time they select the tool.)

6.  Configure settings for releasing roster information. Keep in mind that some external solutions may require this information.

a.  Check the box for Send Names to the External Tool if you want the CLE to release user names of all site members to the external solution.

b.  Check the box for Send Email Addresses to the External Tool if you want the CLE to release email addresses of all site members to the external solution.

7.  Provide additional launch parameters by entering the appropriate information in the Custom Launch Parameters box.

The format for a parameter should be the following:

          keyword=value

You can provide multiple parameters by entering each parameter on a separate line or separating parameters with a semicolon.

8.  To complete the configuration of the tool and launch the external solution, click the Update Options button.

•      If you opted to have the external solution open in a new tab/window, you’ll be taken to the new tab/window. On the tool page, the CLE displays a message that the solution has opened in a new window, along with a link for launching the solution.

•      If you selected the Debug Launch setting, you’ll first see a display of the external solution’s URL and parameters, either within the site or in a new tab/window, depending on the setting. Once you have reviewed this information, click the Press to continue to external tool button.

The external solution will then be displayed, either within the site or in a new tab or window.

Users can access the solution by selecting the appropriate tool name in the tool menu.
