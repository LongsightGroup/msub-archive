Adding Participants

To add participants to your site, access the Site Editor tool and follow these steps:

1.  On the Site Editor home page, click Add Participants in the menu bar. The CLE displays the Add Participants page.

2.  Add one or more participants to the site.

•      To add participants from within your institution, enter each person's user name or email address in the Other Official Participants box. Enter one name or address per line.

•      To add participants who are not from your institution, enter each person’s email address in the Non-Official Participants box. Enter each address on a separate line (you may also add a name after an address, in this format: email address, last name, first name).

3.  Assign a role to participants. To assign all of these participants to the same role, select Assign all participants to the same role. Otherwise, select Assign each participant a role individually.

4.  To set the status for all of these participants, select either Active or Inactive. If you set the status to “inactive,” the participants will not be able to view the site or receive emails sent to the site's e-mail alias. 

5.  Click Continue. What you see next depends on whether you’re assigning all participants to the same role, or assigning each participant to a role individually.

•      If you’re assigning all participants to the same role, the CLE displays a list of site roles. Select the appropriate role.

•      If you’re assigning each participant to a role individually, the CLE displays a dropdown menu for each participant. From these dropdown menus, select the appropriate roles for participants.

6.  Click Continue. The CLE displays an email notification page:

On this page, select an email notification option for the new participants.

7.  To continue adding the participants, click Continue. The CLE displays a confirmation page:

8.  To finish adding new participants, click Finish.

The CLE once again displays the Site Editor home page, where you’ll see the new participants listed.

Warning: If your institution’s CLE instance is integrated with a student information system (SIS), the SIS process may override the add/drop operations you perform in the Site Editor tool. Users in the Guest role are typically not affected by this automated add/drop.
