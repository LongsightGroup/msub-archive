Setting up Turnitin

If your CLE instance has been configured for integration with Turnitin, you can opt to send assignment submissions to Turnitin for originality checking.

Note: The Turnitin integration is an optional feature that requires additional configuration of the CLE and an institutional account with Turnitin.

When using Turnitin, keep in mind the following:

•      Turnitin will only check attachments. It will not check inline submissions.

•      Each submission should include only one attachment. Submissions with multiple attachments will not be processed. To ensure that all submissions for an assignment are checked, select Single Uploaded File only from the Student Submissions menu when creating or editing the assignment.

•      If you allow resubmissions, each resubmission will be sent to Turnitin for originality checking. By default, the CLE provides access to the originality report for the last resubmission, but you may view reports for other submissions from within Turnitin.    

•      An attachment must be in MS Word, PDF, HTML, RTF, or plain text format, and it should always include the appropriate file extension (e.g., “.docx”).  

•      The speed of report processing will depend on the job schedule set by your CLE administrator.   

To set up Turnitin for an assignment, follow these steps.

1.  When creating or editing the assignment, scroll to Turnitin Service, which you’ll find displayed above the Grading settings:

2.  Check the box for Use Turnitin. The CLE displays additional settings, as well as guidelines for submission.

  

3.  For the Submit papers to the following repository setting, choose whether to have student submissions added to a repository that is used for originality checking:

•      None—submissions are not added to a repository.

•      Standard Paper Repository—submissions are added to the Turnitin repository of student work.

•      Institution Paper Repository—submissions are added to your institution’s repository of student work.

4.  For the Generate originality reports setting, choose when originality reports will be generated.

•      Select Immediately if you want reports to be generated as soon as students submit.

•      Select On Due Date if you want reports to be generated on the assignment due date.

5.  If you want to provide students with access to originality reports for their submissions, check the box for Allow students to view report.

6.  For the Check originality against setting, choose the content against which submissions will be checked for originality, by selecting any or all of the following:

•      Turnitin paper repository (Turnitin database of student work)

•      Current and archived internet

•      Periodicals, journals, and publications

•      Institution-specific repository (database of student work from your institution)

Once you’ve selected Turnitin settings, proceed as appropriate for creating or editing the assignment. For instructions on accessing originality reports for an assignment, see Viewing Turnitin Reports.
