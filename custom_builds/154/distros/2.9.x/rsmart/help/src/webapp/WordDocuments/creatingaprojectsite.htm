Creating a Project Site

The following diagram shows the eight steps for creating a project site:

Before you begin to create a project site, be sure to gather the following information.

Item

Description

Site Title

The name of your site. Required.

Site URL Alias

This entry is optional. You may want to use a site URL alias that is more user-friendly than the default, which includes the site ID.

Site description

To be displayed in the site home page.

Short Description (up to 80 characters long)

To be displayed in a public list of sites and on your Projects home page.

Icon URL

URL for the description of the project or a reference to another web page you'd like to use in place of the text-based site description.

Site Contact Name

First and last name of the person responsible for the site.

Site Contact Email

Email address of the person responsible for the site.

Tools

Tools you want to add to the site. If you base your site on a template, the standard tools for that template will be added automatically, but you can change them after creating your site.

Site from which you want to re-use material

Select another site or sites you own, from which you would like to re-use material.

Site email address for email archive (required if Email Archive Tool is selected).

Unique email address for the course.

Title and URL for News (optional if News Tool is selected).

May include more than one title and URL

Title and URL for Web Content (optional if Web Content Tool is selected).

May include more than one title and URL

Site Access: Public site?

Choose whether to make your site accessible to all users with a CLE login ID and password.

Global Access: Joinable? If yes, which role can join the site?

 

After collecting the necessary information, follow these steps.

1.  Choose one of these options:

•      If you have a good understanding of the tools available for use in project sites and are willing to choose tools in a later step of the site creation process, you may build your own project site. For further instructions on building your own site, proceed to step 3.

•      If you are new to the CLE tools or do not want to select tools for this site from a list, you may create a site from a template. A template eliminates the need for you to understand the full range of tools by providing a set of appropriate tools for your new site. For further instructions on using a template, proceed to the next step.

Note: Regardless of which option you choose, you may add and delete tools at any time after creating your site.

2.  To create a site using a template, first click Create site from template. Then, proceed with the following additional steps.

a.  Select the appropriate project site template from the dropdown list.

b.  In the Site title box, enter a title for your project site.

c.  If you would like to copy the list of site participants and/or tool content from the site being used as a template, click Copy users and/or Copy content.

d.  Click Continue. The CLE displays the Site Setup home page, where you will see your new project site included in the list of sites.

3.  To create your own site, select Build your own site and then click project site. Next, click Continue. The CLE displays the Project Site Information screen.

4.  Enter project information, as follows:

•      In the Site Title box, enter a title for the site (maximum 19 characters).

Note: The Site Editor tool allows you to enter more than 19 characters for the site title. After creating the site in the Site Setup tool, you may want to use the Site Editor to create a longer title for your site (up to 50 characters). For more information on using the Site Editor, see Site Editor.

•      In the Site URL Alias box, optionally enter a site URL alias. This alias can be a more user-friendly URL than the default, which includes the site ID.

•      In the Description box, enter a description of the site. Use the icons and pulldown lists to format the text, as appropriate. For information on using these options, see Using the Rich Text Editor.

•      In the Short Description box, enter a shorter description of the site (maximum 80 characters). The short description is displayed in the public list of sites and on your Courses home page.

•      If desired, in the Icon URL box, enter a URL for the description of the project or a reference to another web page you'd like to use in place of the text-based site description.

•      In the Site Contact Name box, enter the first and last name of the person responsible for the site.

•      In the Site Contact Email box, enter the email address of the person responsible for the site.

5.  Click Continue. The CLE displays the Project Site Tools screen.

6.  Check appropriate boxes to select tools to be used on your site. Or, to select all of the tools, check the box for All tools at the top of the list.

7.  If you plan to reuse material from other sites you own, proceed to the next step. Otherwise, proceed to step 9.

8.  To reuse material from other sites you own (such as a site for this same class for a previous term):

a.  Select Yes, from these sites.

b.  Scroll through the list and select the site(s) from which you wish to reuse material.

c.  Click Continue. The CLE displays another screen.

     Click on the appropriate check boxes to choose the tools from which you want           to reuse material. Then, click Continue. Proceed to step 10.

9.  If you are not reusing materials from other sites you own, select No, thanks. Then, click Continue.

10. If you are using a tool that requires more information, the CLE displays the Add Multiple Tool Instances screen. If you are not, the CLE displays the Course Site Access screen. In this case, proceed to step 11.

     On the Add Multiple Tool Instances screen, enter the requested information      for each tool.

     The type of information requested will vary, depending on the tools you have   selected. For example:

•      If you have selected the Email Archive tool, you must select a unique archive email address for the site.

•      If you have selected the News or Web Content tool, you must enter a title and a URL.

•      To add additional instances of the News or Web Content tool, select the number of tools of each type from the appropriate More? dropdown list and provide all appropriate information for the additional tools.

     To proceed, click Continue. The CLE displays the Project Site Access screen.

11. On the Project Site Access screen, choose how you would like the site to be accessed by users.

     Choose from these options:

     Site Status

•      To publish your site so it is available to site participants, select Publish site.

•      To keep your site unpublished so it is available only to you and other site maintainers (such as CLE Functional Administrators), select Leave as Draft – accessible only to site maintainers.

     Site Visibility

•      To keep your site visible only to site participants, select Private.

•      To display your site in a list of public sites, so users may search for it on the login page and also access any content on the site you have made public, select Display in public site list.

     Global Access

•      To make your site accessible only to site participants, select Limited to whom I add manually, or through automatic roster updates.  

•      To make your site accessible to all users with a CLE login ID and password, select Allow anyone to join the site with valid login id. Then, use the pulldown list to select an appropriate role for users who choose to join the site.

12. Next, click Continue. The CLE displays the Confirm Your Project Site Setup screen.

 

13. Review the information on the confirmation screen. To finish creating the site, click Create Site. The CLE initially displays the following message.

     The CLE then displays the Site Setup home page, with your new project site    included in the list of sites.
