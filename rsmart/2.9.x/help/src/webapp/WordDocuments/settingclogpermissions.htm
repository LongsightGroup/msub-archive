Setting Clog Permissions

If you are a site owner, you may specify how site participants use the Clog tool by setting user permissions.

To set permissions, click on Permissions in the Clog menu bar. The CLE displays the Permissions page.

For each role in the site, you’ll find two sets of identical permissions—one set for blog posts, and another set for comments. See the following table for descriptions of these permissions:

 

Permission

Description

create

Users can create blog posts/comments.

read.any

Users can view blog posts/comments by all site participants.

update.any

Users can edit blog posts/comments by all site participants.

update.own

Users can edit only their own blog posts/comments.

delete.any

Users can delete blog posts/comments by all site participants.

delete.own

Users can delete only their own blog posts/comments.

After setting all appropriate permissions, click Save. If you do not want to save your changes, click Cancel. In either case, the CLE then displays the home page.
