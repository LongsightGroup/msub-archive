Site Setup > Creating a Project Site

Building Your Own Project Site

To build your own project site in the CLE, access My Workspace and select the Site Setup tool. Then, follow these steps.

1.  From the Site Setup home page, click New in the menu bar. (Note: if you do not see this button, you do not have permission to create new sites.)

The CLE displays the Site Type page.

2.  On this page, click on the project site radio button. (Note: if you do not see a project button, you do not have permission to create project sites.)  

Then, click Continue. The CLE displays the Project Site Information page.

3.  Provide information for the project site:

•      In the Site Title box, enter a title for the site (maximum 19 characters).

•      Depending on how your CLE instance has been configured, you may be able to provide a site URL. In the Site URL box, optionally enter a more user-friendly URL than the default, which includes the site ID.

•      From the Site Language menu, optionally select a different language for your site. By default, the site will use the language for which your CLE instance has been configured.

•      In the Description box, use the rich text editor to provide a site description that will display on the site’s home page.

•      In the Short Description box, enter a shorter site description (maximum 80 characters) that users can view if the site is displayed in the list of public sites.

•      In the Icon URL box, you may optionally enter a URL for an image that will display above your site’s tool menu.

•      In the Site Contact Name box, enter the first and last name of the person responsible for the site.

•      In the Site Contact Email box, enter the email address of the person responsible for the site.

4.  Click Continue. The CLE displays the Project Site Tools page. On this page, check appropriate boxes to select tools to be used on your site. Or, to select all of the tools, check the box for All tools at the top of the list.

5.  If you don’t want to reuse materials from other sites you own, select No, thanks and then click Continue. If you do want to reuse material from other sites you own:

a.  Select Yes, from these sites.

b.  Scroll through the list and select the site(s) from which you wish to reuse material.

c.  Click Continue. The CLE displays another screen listing tools for which you may reuse material. Check boxes to choose the appropriate tools. Then, click Continue again.  

Your next step will depend on whether any tools in your site require further configuration:

•      If one or more tools require further configuration, the CLE displays the Add Multiple Tool Instances page. Proceed to step 6.

•      If no tools require configuration, the CLE displays the Project Site Access page. Proceed to step 7.    

6.  On the Add Multiple Tool Instances page, enter the appropriate information for each tool.

The type of information you provide will vary, depending on the tools you have selected:

•      If you have selected the Email Archive tool, you must select a unique archive email address for the site.

•      If you have selected the Basic LTI or Lesson Builder tool, you may enter a different tool title.

•      If you have selected the News or Web Content tool, you may enter a different tool title as well as enter a URL.

You may also add additional instances of the Basic LTI, Lesson Builder, News and Web Content tools to your site. To do so, select the number of instances for a tool from the dropdown list and provide the appropriate information.

     Click Continue. The CLE displays the Project Site Access page.

7.  On the Project Site Access page, choose how you would like the site to be accessed by users.

     Choose from these options:

     Site Status

•      To publish your site so it is available to site participants, select Publish site.

•      To keep your site unpublished so it is available only to you and other site maintainers (such as CLE Functional Administrators), select Leave as Draft – accessible only to site maintainers.

Site Visibility

•      To keep your site visible only to site participants, select Private.

•      To display your site in a list of public sites, so users may search for it on the login page and also access any content on the site you have made public, select Display in public site list.

Global Access

•      To make your site accessible only to site participants, select Limited to whom I add manually, or through automatic roster updates.   

•      To make your site accessible to all users with a CLE login ID and password, select Allow anyone to join the site with valid login id. Then, use the dropdown list to select an appropriate role for users who choose to join the site.

8.  Click Continue. The CLE displays the Confirm Your Project Site Setup page. Review the information on this page. To finish creating the site, click Create Site.

The CLE then displays the Site Setup home page, where you’ll see your new project site included in the list of sites.
